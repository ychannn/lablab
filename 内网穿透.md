# 内网穿透（让老师在外网访问）

本地没有公网 IP 时，可用内网穿透把后端和前端暴露出去，老师通过生成的网址即可访问。

## 1. 启动后端

在项目根目录（`d:\lablab`）执行：

```bash
mvn spring-boot:run
```

后端默认运行在 **8080** 端口。

## 2. 用 ngrok 暴露三个服务

需要三个隧道：后端 8080、门户前端 5173、管理端 5174。

- 安装 [ngrok](https://ngrok.com/) 并登录，在终端执行 `ngrok config add-authtoken <你的token>`。
- 开三个终端，分别执行（免费版一次只能一个隧道，可先只开后端+一个前端做演示）：

```bash
# 终端1：后端
ngrok http 8080
# 记下生成的 https 地址，如 https://xxxx-xx-xx.ngrok-free.app
```

```bash
# 终端2：门户前端（Vite 默认 5173）
cd frontend && npm run dev
# 再开一个终端对 5173 做穿透：
ngrok http 5173
# 记下门户的 https 地址
```

```bash
# 终端3：管理端（admin 默认 5174）
cd admin && npm run dev
# 再开一个终端对 5174 做穿透：
ngrok http 5174
# 记下管理端的 https 地址
```

若使用其他工具（如 cpolar、frp），同理：后端 8080、门户 5173、管理端 5174。

## 3. 让前端请求指向“后端的外网地址”

前端需要把 API 请求发到**后端穿透后的地址**，否则老师浏览器会去请求老师本机的 localhost。

把**后端的 ngrok 地址**设为环境变量再启动前端（替换成你实际的地址）：

**Windows PowerShell：**

```powershell
$env:VITE_API_BASE="https://xxxx-xx-xx.ngrok-free.app"
cd d:\lablab\frontend
npm run dev
```

```powershell
$env:VITE_API_BASE="https://xxxx-xx-xx.ngrok-free.app"
cd d:\lablab\admin
npm run dev
```

**Windows CMD：**

```cmd
set VITE_API_BASE=https://xxxx-xx-xx.ngrok-free.app
cd d:\lablab\frontend
npm run dev
```

**Linux / macOS：**

```bash
export VITE_API_BASE=https://xxxx-xx-xx.ngrok-free.app
cd frontend && npm run dev
```

- 门户、管理端都会读 `VITE_API_BASE`，不设则默认 `http://localhost:8080`。
- 修改环境变量后需**重新执行** `npm run dev` 才会生效。

## 4. 给老师的访问方式

- **门户（对外展示）**：把「门户前端的 ngrok 地址」发给老师，例如 `https://yyyy-yy-yy.ngrok-free.app`。
- **管理后台**：把「管理端的 ngrok 地址」发给老师，例如 `https://zzzz-zz-zz.ngrok-free.app`。老师用管理员账号登录即可。

## 5. 注意

- 免费 ngrok 每次重启会变域名，需要重新把新地址发给老师，并重新设 `VITE_API_BASE` 再启动前端。
- 后端、两个前端可以分开启动，不要求“一起点运行”；只要后端先起，再起前端并设好 `VITE_API_BASE` 即可。
